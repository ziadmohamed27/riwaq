# QA Checklist — رواق MVP

هذه القائمة مخصصة لاختبار النسخة الحالية من رواق قبل الانتقال إلى صفحات الأدمن أو النشر.

## 1) المصادقة وحماية المسارات
- [ ] المستخدم غير المسجل يُعاد توجيهه من `/cart` إلى صفحة الدخول عند الحاجة.
- [ ] المستخدم غير المسجل يُعاد توجيهه من `/checkout` إلى صفحة الدخول.
- [ ] المستخدم غير المسجل لا يصل إلى `/account/*`.
- [ ] المستخدم غير المسجل لا يصل إلى `/seller/*`.
- [ ] المستخدم المسجل كعميل لا يصل إلى صفحات البائع قبل قبول طلبه.

## 2) الكتالوج والمنتجات
- [ ] صفحة `/marketplace` تعرض المنتجات النشطة فقط.
- [ ] المنتجات التابعة لمتاجر غير نشطة لا تظهر.
- [ ] البحث العربي يعمل مع التطبيع: أ/إ/آ -> ا ، ة -> ه ، ى -> ي.
- [ ] فلترة التصنيف تعمل وتنعكس على الرابط.
- [ ] فلتر السعر يعمل بشكل صحيح.
- [ ] حالة عدم وجود نتائج تظهر برسالة واضحة.
- [ ] صفحة `/product/[slug]` تعرض المنتج الصحيح.
- [ ] المنتج غير الموجود يؤدي إلى `notFound()`.
- [ ] حالة المخزون تظهر بشكل صحيح: متوفر / آخر X قطع / نفد المخزون.

## 3) السلة
- [ ] إضافة منتج إلى السلة تعمل للمستخدم المسجل.
- [ ] المستخدم غير المسجل يُعاد توجيهه لتسجيل الدخول عند الإضافة.
- [ ] تعديل الكمية ينعكس مباشرة.
- [ ] حذف عنصر من السلة يعمل.
- [ ] إفراغ السلة يعمل.
- [ ] إضافة منتج من متجر مختلف تفتح نافذة التعارض.
- [ ] تأكيد التعارض يمسح السلة القديمة ويضيف المنتج الجديد.
- [ ] السلة الفارغة تعرض Empty State مناسبة.

## 4) إتمام الطلب
- [ ] صفحة `/checkout` بدون سلة تعيد التوجيه إلى `/cart`.
- [ ] العناوين تظهر وتُحدَّد البطاقة الافتراضية تلقائيًا.
- [ ] زر "إضافة عنوان جديد" يذهب إلى `/account/addresses?redirect=/checkout`.
- [ ] بعد إضافة عنوان جديد، يرجع المستخدم إلى `/checkout` تلقائيًا.
- [ ] الضغط على "تأكيد الطلب" يستدعي `create-order` بنجاح.
- [ ] يتم إنشاء `orders` و `order_items` و `order_status_history`.
- [ ] `carts.status` تتحول إلى `checked_out`.
- [ ] المخزون يخصم عند نجاح الطلب.
- [ ] صفحة النجاح تعرض `orderNumber` الصحيح.

## 5) حساب العميل
### العناوين
- [ ] إضافة عنوان جديد تعمل.
- [ ] تعديل عنوان موجود يعمل.
- [ ] حذف عنوان يعمل.
- [ ] تعيين عنوان افتراضي يعمل.
- [ ] إذا لا توجد عناوين، تظهر Empty State مناسبة.

### الطلبات
- [ ] `/account/orders` تعرض طلبات المستخدم الحالي فقط.
- [ ] كل بطاقة تعرض رقم الطلب والتاريخ والإجمالي والحالة وعدد العناصر.
- [ ] `/account/orders/[orderNumber]` تعرض snapshots الصحيحة للمنتجات والعنوان.
- [ ] محاولة فتح طلب لا يخص المستخدم تُظهر 404 أو تمنع الوصول.

## 6) البائع
### onboarding
- [ ] المستخدم يستطيع إرسال طلب بائع جديد.
- [ ] إذا كان لديه طلب pending لا يستطيع إرسال طلب جديد.
- [ ] إذا كان لديه طلب approved لا يستطيع إعادة التقديم.
- [ ] صفحة الحالة تعرض: pending / approved / rejected بشكل صحيح.
- [ ] سبب الرفض يظهر إن وجد.

### مساحة البائع
- [ ] `/seller` تعرض الإحصاءات الأساسية.
- [ ] بطاقة "طلبات جارية" تنقل إلى طلبات مفلترة بشكل صحيح.
- [ ] بطاقة "مخزون منخفض" تنقل إلى المنتجات ذات المخزون المنخفض فقط.

### المنتجات
- [ ] البائع يرى منتجات متجره فقط.
- [ ] إضافة منتج جديد تعمل.
- [ ] تعديل منتج يعمل.
- [ ] أرشفة منتج تعمل.
- [ ] رفع الصور يعمل بدون كسر حفظ المنتج.
- [ ] حالات المنتج تظهر موحدة: مسودة / نشط / مؤرشف.

### الطلبات
- [ ] البائع يرى طلبات متجره فقط.
- [ ] فلتر `status=active` يعرض confirmed + processing + shipped.
- [ ] تحديث الحالة يعرض فقط الانتقالات المسموح بها.
- [ ] نجاح التحديث ينعكس بعد `router.refresh()`.

## 7) الرسائل والنصوص
- [ ] حالات الطلب موحدة في كل الصفحات.
- [ ] لا توجد صيغ متضاربة مثل "جاري التجهيز" و"قيد التجهيز" في نفس المنتج.
- [ ] لا توجد صيغ متضاربة مثل "مسترجع" و"مسترد".
- [ ] النصوص العربية RTL ومحاذاتها صحيحة.

## 8) جاهزية ما قبل النشر
- [ ] `.env.example` محدث ويحتوي كل المتغيرات اللازمة.
- [ ] `README.md` واضح لخطوات التشغيل.
- [ ] `DEPLOYMENT_CHECKLIST.md` تمت مراجعته.
- [ ] جميع الروابط الرئيسية تعمل بلا 404 غير مقصود.
- [ ] لا توجد imports مكسورة في الملفات المضافة حديثًا.
