# UI Copy Guide — رِواق

هذا الملف يثبت الصياغة العربية المعتمدة في واجهة رِواق حتى تبقى النصوص متسقة بين صفحات العميل والبائع.

## نبرة البراند
- عربية واضحة
- أنيقة وبسيطة
- مباشرة من دون مبالغة
- احترافية لكن دافئة

## تسميات الحالات المعتمدة

### حالات الطلب
- pending → قيد الانتظار
- confirmed → مؤكد
- processing → قيد التجهيز
- shipped → تم الشحن
- delivered → تم التسليم
- cancelled → ملغي
- refunded → مسترد

### حالات المتجر
- active → نشط
- suspended → موقوف
- closed → مغلق

### حالات طلب البائع
- pending → قيد المراجعة
- approved → تمت الموافقة
- rejected → مرفوض

### حالات المنتج
- draft → مسودة
- active → نشط
- archived → مؤرشف

### حالات الدفع
- pending → بانتظار الدفع
- paid → مدفوع
- refunded → مسترد
- failed → فشل الدفع

## رسائل المخزون
- stock = 0 → نفد المخزون
- stock <= 5 → آخر X قطع
- stock > 5 → متوفر

## أمثلة صياغة معتمدة

### نجاح
- تم حفظ البيانات بنجاح.
- تم استلام طلبك بنجاح.
- تم تحديث الحالة بنجاح.

### أخطاء
- تعذّر إتمام العملية حاليًا.
- تعذّر حفظ البيانات. حاول مرة أخرى.
- لا يمكن الوصول إلى هذا الطلب.

### Empty states
- لا توجد طلبات حتى الآن.
- لا توجد منتجات مطابقة حاليًا.
- لم تُضف أي عناوين بعد.

## ملاحظات
- تجنب استخدام صيغتين مختلفتين لنفس الحالة مثل:
  - جاري التجهيز / قيد التجهيز
  - مسترجع / مسترد
  - تم التأكيد / مؤكد
- استخدم القيم المركزية من:
  - `lib/utils/arabic.ts`
